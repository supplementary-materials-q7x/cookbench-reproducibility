Prerequisites

Operating system

Windows with a primary display configured at 100% scaling. Place the game window on the primary display for consistent screen capture.

Game dependency

Cooking Simulator plus Food Network DLC, installed through the official distributor. Use a local Sandbox session in the Classic kitchen scene.

Software environment

Python 3.10 with Conda; an editor such as VS Code is optional. Install MelonLoader version 0.5.7 from the official source.

Game-side installation

1
Install the loader. Install MelonLoader 0.5.7 for the locally licensed Cooking Simulator installation.
2
Install the integration files. Copy CS_CamDump.dll to <Cooking Simulator>/Mods/. Copy item_name_mapping.txt and object_en_ch_mapping.txt to <Cooking Simulator>/UserData/.
3
Verify the integration. Start the game and press F12 to start or stop the real-time scene scanner. Pick up and move an object: its colored position marker should move with it. Press F10 to show or hide the position/diagnostic visualization; it is useful for setup validation and can remain hidden during a timed episode.
4
Fix the scenario. Start a new Sandbox session in the Classic kitchen scene and apply the documented game settings before each run.

Resume, rollback, and batch execution

# Resume the most recent compatible run and refresh its memory state
python epm/scripts/run_episode_dashboard.py \
  --config epm/configs/pipelines/planner_executor.json \
  --dish-id 1 --resume --refresh-memory [--run-name <RUN_NAME>]

# Resume a named run at an explicit step, or roll back relative to its current checkpoint
python epm/scripts/run_episode_dashboard.py \
  --config epm/configs/pipelines/react.json \
  --resume --run-name <RUN_NAME> --resume-step-id 80
python epm/scripts/run_episode_dashboard.py \
  --config epm/configs/pipelines/react.json \
  --resume --run-name <RUN_NAME> --rollback-steps 5

# Batch runs
python epm/scripts/run_episode_dashboard.py \
  --config epm/configs/pipelines/reflexion.json \
  --dish-range 1-20,41-60 --auto-chain
python epm/scripts/run_episode_dashboard.py \
  --config epm/configs/pipelines/reflexion.json \
  --resume --refresh-memory --auto-chain --dish-range 1-131
python epm/scripts/run_episode_dashboard.py \
  --config epm/configs/pipelines/reflexion.json \
  --dish-range 1-131 --auto-chain --planner-model gpt-5-mini
python epm/scripts/run_episode_dashboard.py \
  --config epm/configs/pipelines/planner_executor.json \
  --dish-range 40,41,83,88 --resume --refresh-memory --auto-chain

With --resume --dish-range 40,41,83,88, the runner first identifies the latest compatible run and then continues from that dish's position in the listed sequence; it does not unconditionally restart at dish 40.

Restart and runtime overrides

# Restart the game environment for a fresh scene when the evaluation UI is available
python epm/scripts/run_episode_dashboard.py \
  --config epm/configs/pipelines/planner_executor.json \
  --dish-id 102 --restart-env

# Override a nested runtime parameter for one invocation
python epm/scripts/run_episode_dashboard.py \
  --config epm/configs/pipelines/planner_executor.json \
  --dish-id 1 --runtime.force_submit_step_threshold 300

Evaluation feedback capture

# After manual submission opens the in-game evaluation UI, write feedback to the latest run
# First set the script's userdata-root argument/path to the local game UserData directory.
python ./epm/scripts/dump_recipe_feedback.py

Before closing the evaluation UI, verify that the recorded dish name is correct. A run should retain its trajectory, configuration snapshot, runtime log, screenshots, score, and textual feedback such as flavors, technique, temperature, and unwantedProducts.

Reproduction checklist

Before running

Record OS, game version/DLC, loader version, integration-package checksum, screen scaling, configuration revision, and model/provider access date.

For every episode

Save the dish ID, framework/configuration, prompt-ablation choice if applicable, trajectory, runtime log, screenshot evidence, and evaluation output.

When reporting

Separate released author-created artifacts from commercial simulator dependencies and external model APIs. Report provider defaults and unresolved randomness rather than implying deterministic replay.
